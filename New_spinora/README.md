# SpinoraBot - Розыгрыши "Колесо фортуны" для Telegram

## 🎯 Описание

SpinoraBot - это бот для создания и управления розыгрышами "Колесо фортуны" в Telegram каналах. Проект включает Telegram бота, мини-приложение и backend сервер.

## 🚀 Быстрый старт

### 1. Настройка окружения

Скопируйте пример конфигурации:
```bash
cp .env.example .env
```

Отредактируйте `.env` файл:
```env
# Обязательные параметры
BOT_TOKEN=ваш_токен_бота_от_@BotFather
PUBLIC_WEBAPP_URL=https://ваш-домен.ngrok.io

# Опциональные параметры
PORT=3000
DEV_AUTH_BYPASS=1  # Для локальной разработки
```

### 2. Запуск сервисов

**Linux/Mac:**
```bash
chmod +x start_services.sh
./start_services.sh
```

**Windows:**
```cmd
start_services.bat
```

### 3. Доступ к приложению

После запуска сервисы будут доступны по адресу:
- Мини-приложение: `http://localhost:3000`
- Логи: `logs/web.log` и `logs/bot.log`

## 🎮 Основные функции

### Бот (/start)
- 🧩 Запустить мини-приложение
- 📝 Создать пост (фото/видео/документ/текст)
- 📣 Мои каналы

### Мини-приложение
Мастер создания розыгрыша из 4 шагов:
1. **Тип розыгрыша** - Колесо фортуны / Открытие кейса
2. **Настройки** - Название, язык, выбор поста
3. **Каналы** - Список каналов для публикации
4. **Призы** - Настройка призов с количеством

## 🛠 Разработка

### Режим разработки
Установите `DEV_AUTH_BYPASS=1` в `.env` для обхода авторизации Telegram.

### API Endpoints
- `GET /api/health` - Проверка состояния
- `GET /api/me` - Информация о пользователе
- `GET /api/posts?scope=drafts` - Список постов
- `POST /api/giveaways` - Создание розыгрыша

### Хранилище данных
Используется JSON файл `data/storage.json` с атомарной записью:
```json
{
  "users": {},
  "post_drafts": {},
  "giveaways": {},
  "counters": {
    "post_id": 0,
    "giveaway_id": 0
  }
}
```

## 🐛 Решение проблем

### Бот не запускается
Проверьте:
- Наличие `BOT_TOKEN` в `.env`
- Установленные зависимости: `pip install -r bot/requirements.txt`

### Web сервер не запускается
Проверьте:
- Свободен ли порт 3000
- Установленные зависимости: `cd web && npm install`

### Ошибки авторизации
Для локальной разработки:
- Установите `DEV_AUTH_BYPASS=1`
- Используйте тестовые данные пользователя
